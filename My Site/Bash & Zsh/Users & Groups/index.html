Command Description
cat /etc/passwd - file contains all users in the system
- each entry represents a user (userName : x

- the 'x' represents the encrypted password

: uid : gid : fullName : homeDirPath : loginShellPath)
cat /etc/group - file contains all groups in the system
- each entry represents a group (groupName : x

- the 'x' represents the encrypted password

: gid : groupMember,...)
|
id user - returns the current (or passed) user's userId, groupId and groups the user is part of
groups user ... - returns the groups of the specified user(s)
|
adduser [options] userName - (no options) adds the specified user to the system (also creates user home directory and primary group)
[options]
- (--uid uid) specifies the user id of the user
- (--system) creates a system user
- (--no-create-home) does not create home directory for the user
- (--home dirPath) sets the home directory for the user
adduser userName groupName - adds the user to the specified group
deluser [options] userName - (no options) removes the specified user from the system
[options]
- (--remove-all-files) removes all files which belongs to the user
- (--remove-home) remove the home directory of the user
deluser userName groupName - removes the user from the specified group
|
groupadd [options] groupName - (no options) creates the specified group
[options]
- (--gid gid) sets the gid of the group
- (--system) creates a system group
groupdel [options] groupName - removes the specified group
[options]
- (-f) also removes primary group
groupmod --gid gid groupName - sets the GID of the specified group
groupmod --new-name newName groupName - changes the specified group's name
|
passwd [options] userName - changes the password for the current or for the specified user
[options]
- (-d) deletes the specified user's password (deleted password works when login but on terminal the user still needs password [TESTED: 2020-04-11])
usermod [options] userName - modifys the specified user account
[options] (at least one is required)
- (-d dirPath) sets the home directory for the user
- (-uid uid) sets the user id of the user
- (-gid gid) sets the primary group of the user
usermod --login newName userName - renames the specified user
|
su userName - switches the user on the terminal (sudo su = switches to the super user)
|
sudo cmd - executes the command as root user
|
w | users - list the currently logged-in users (w provides more information)
