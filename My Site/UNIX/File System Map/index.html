File System Map

Updated ( 2020-03-10 / 2020-03-30 )

MySite> UNIX> File System Map

Folder Description
/ - root directory
|
/bin - (binary) directory holds essential binaries (booting, system emergency, single user mode, etc..)
|
/boot - (boot) directory for Linux boot loader files
|
/cdrom - (CD-ROM) directory for cd-rom devices
|
/dev - (devices) directory for device files (in Linux every device is depicted as a file, like mouse, keyboard, etc...)
|
/etc - (etcetera) directory for system-wide configuration files, these configurations are valid for all users
/enviroment - file contains system-wide environment variables
/group - file contains all groups in the system
- each entry represents a group (groupName : x

- the 'x' represents the encrypted password

: gid : groupMember,...)
/hosts - file contains local DNS mapping (local domain to IP address mapping)
- (column1 = ip address) (column2 = domain) (column3 = domain alias)
/os-release - file contains operating system informations (like linux distro, version, etc...)
/passwd - file contains all users in the system
- each entry represents a user (userName : x

- the 'x' represents the encrypted password

: uid : gid : fullName : homeDirPath : loginShellPath)
/resolv.conf - file contains DNS server IPs (thse servers are requested for DNS resolution)
/ssh - directory contains system whide ssh config items
/ssh_config - file contains ssh system whide configurations
/sshd_config - file contains ssh server configurations

PasswordAuthentication yes|no: enable|disable password authentication for all users (Default: yes)

AllowUsers user, ...: only the listed users are allowed to access the system (white list)

DenyUsers user, ...: listed users are allowed to access the system (black list)

Match User user: below listed directives are valid for user only

ex: (password authentication disabled except for user remote) 

      PasswordAuthentication no
      
      Match User remote
          PasswordAuthentication yes
/sudoers - file contains users and groups with administrator priviledges (sudo must be installed)
|
/home - (home) directory for user(s)
- each user has its own home folder and cannot access other home folders without superuser permissions
- holds the current user application configurations setting caching
/user
/.bash_aliases - file contans user command aliases (ex:)

alias aliasCmd=cmd

...

EX:

alias test="ls | grep test" (list all test files in the current directory)


- must be read and executed from ~/.bashrc 
    if [ -f ~/.bash_aliases ]; then
    . ~/.bash_aliases
    fi
/.bashrc|.zshrc - file contains commands that are executed every time when a new shell (bash or zshell) is opened (each line is a new command)
/.profile|.zprofile - file contains commands that are executed when a log in shell is opened

- when a user logs in with user and password so the shell initializes the user's environment

(each line is a new command)
- its mostly used to setup user specific environment variables
/.ssh - directory contains ssh private & publick keys
- usable only by the user (even a separate user part of the folder's group has no access for security reasons)
/config - file contains ssh client configurations
- maps host names to connections
    Host hostName
      Hostname host|ip
      Port port
      User user
    
    ...

- cmd to connect ssh hostName

/known_hosts - file contains already connected connection fingerprints (this is how the ssh client 'remembers' connections)
- if the remote host changes the user is warned (protects against man-in-the middle attacks)
/known_hosts.old - one version older than known_hosts (backup file)
/authorized_keys - file contains ssh publick keys for the ssh server (daemon)
- this is where the ssh server keeps its public keys
|
/libnr - (libraries) directory holds system libraries
|
/media - (removable media) directory for external storage devices are mounted her by the OS (like USB sticks, SD cards, etc...)
|
/mnt - (mount) directory for manually mounted storage and partitions
|
/opt - (optional) directory for manually installed softwares which are not part of the default installation
|
/proc - (process) directory for process and kernel information files, the OS keeps track of running processes and other OS functionalities in files here
|
/root - (root) directory of the superuser
|
/run - (run-time variables) directory for initial runing system files since the last boot (OS creates this directory on every boot up)
|
/sbin - (system binaryes) directory for holding system binary files which are accessible only for the superuser
|
/snap - (snap packages) directory for Ubuntu snap packages
|
/srv - (server) directory for server data is sotred here
|
/sys - (system) directory for kernel configuration files (OS creates this directory on every boot up)
|
/tmp - (temporary) directory for temporary files stored by applications
|
/usr - (user) directory for user applications (non-essential for system operations)
/bin - directory user binaries (non required by the system)
/share - directory for sharable architecture independend files (icons, fonts, docs, etc..)
/doc - directory for package documentations, configurations, utilities, etc...
|
/var - (variables) directory for data files, log files, cache data, waiting tasks, etc...
/log - directory contains log files
/auth.log - file contains authentication logs (like ssh authentication logs)
/spool/cron/crontabs - directory contains user crontab files

File Acces Modes
Number Symbol Permission Type
0 --- No Permission
1 --x Execute
2 -w- Write
3 -wx Write + Execute
4 r-- Read
5 r-x Read + Execute
6 rw- Read + Write
7 rwx Read + Write + Execute
